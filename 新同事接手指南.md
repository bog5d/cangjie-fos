## 这套系统是干嘛的

**仓颉 FOS**（融资作战操作系统）= 帮我们管理融资流程的内部工具。

核心功能一句话：
- 记录我们在跟哪些机构谈（Pipeline 看板）
- 路演结束后自动生成评分报告和情报摘要
- 机构发来的尽调清单，AI 自动帮你在材料库里找对应文件
- 投后季报模板自动填充（AI 从材料里提取数据填入空格）
- 所有数据本地存，不上云（GitHub 同步是可选功能）

## 第二步：安装运行环境（首次约 10 分钟）

### 需要先装好这两样东西

| 工具 | 用途 | 安装方式 |
|------|------|----------|
| Python 3.11+ | 跑后端 | https://www.python.org/downloads/ ← 安装时勾选 **Add to PATH** |
| uv | Python 包管理器 | 装好 Python 后运行：`pip install uv` |
| Node.js 18+ | 跑前端（可选） | https://nodejs.org/ |

### 配置环境变量

```bash
cd backend
# Windows:
copy .env.example .env
# Mac/Linux:
cp .env.example .env
```

用记事本打开 `backend/.env`，填入：

```
DEEPSEEK_API_KEY=sk-王波给你的key
DASHSCOPE_API_KEY=sk-王波给你的key（没有可以先留空）
FOS_ACCOUNTS=zt001:123456:zt,gk001:123456:gk
```

> ⚠️ **重要**：`FOS_ACCOUNTS` 决定**谁能登录**。格式是 `用户名:密码:团队`，多个账号用英文逗号分隔。  
> 如果只填一个账号，其他账号就无法登录。  
> 如果不填 `FOS_ACCOUNTS`，系统用内置默认账号（zt001/gk001，密码均 123456）。

> **关于 GitHub Token**：系统里有「数据同步到 GitHub」功能，**不填也能正常用所有功能**，Settings 页面 Token 输入框直接留空跳过。

## 系统功能使用说明

### 1. 机构 Pipeline 看板（主页左侧）

**解决什么问题**：记录我们在跟哪些机构推进，每家到了哪个阶段。

**怎么用**：
- 点「➕ 新增机构」→ 填机构名、选阶段、选热度 → 创建
- 点任意机构卡片 → 弹出编辑框 → 修改阶段、记录里程碑
- 成就墙（页面右侧）根据里程碑实时更新统计

**9个里程碑节点**：路演接触 → NDA签署 → 线下交流 → 立项 → 内部尽调 → 外部尽调 → 投决过会 → 协议签署 → 交割

---

### 2. 尽调响应台（顶部「📋 尽调响应」按钮）

支持两种模式（顶部 Tab 切换）：

#### 尽调模式（DD）

**解决什么问题**：机构发来50条尽调清单，AI 帮你在材料库找对应文件。

**三步走**：
1. **Step 1 — 扫描材料库**：填入材料文件夹路径 → 点「开始扫描」→ 等进度条
2. **Step 2 — 上传清单**：粘贴清单文本或上传 Excel/Word/PDF → 填机构名 → 点「解析 & 开始匹配」
3. **Step 3 — 审核确认**：
   - 🟢 绿色高置信 → 「✓ 一键确认」
   - 🟡 黄色 → 看文件名对不对，对了点「✓」
   - 🔴 红色 → 点「📂 替换」手动指定，或点「缺」标记无材料
   - 确认完 → 填导出路径 → 「导出文件夹」→ 生成「缺失清单.txt」

#### 投后模式（季报）

**解决什么问题**：LP/GP 要求填季报，AI 从现有材料里提取数据填入模板空格。

**三步走**：
1. **Step 1 — 扫描材料库**：和尽调模式相同
2. **Step 2 — 上传季报模板**：把需要填写的季报模板文字粘贴进来
3. **Step 3 — 查看草稿 & 导出**：AI 会把每个「【 】」空格的填充值显示出来，点「生成季报初稿」导出 .txt 文件，人工审核后用

---

### 3. 资产台账（页面中部「资产台账」区块）

**解决什么问题**：快速搜索材料库里有没有某类文件。

**怎么用**：在搜索框输入关键词 → 搜索已扫描的材料库

---

### 4. 路演评分工作台（顶部「上传路演录音」入口）

**解决什么问题**：路演录音 → 自动转文字 → AI 分析 → 评分报告，不用手动整理。

**怎么用**：上传录音（mp3/wav/m4a）→ 填机构名和参会者 → 等处理完 → 查看评分报告

> ⚠️ 需要 `DASHSCOPE_API_KEY` 才能使用语音转写。

---

### 5. 路演 AI 教练（顶部「🎤 教练」入口，新功能）

**解决什么问题**：练习路演演讲，AI 实时分析哪些关键要点没讲到、语速如何。

**怎么用**：
- 上传 BP 文本 → AI 提取关键要点 → 进入练习模式
- 上传录音 → AI 评分「要点覆盖率」和「演讲节奏」
- 多轮练习 → 查看进步曲线

---

### 6. 答疑 AI 审问（顶部「❓ 答疑」入口，新功能）

**解决什么问题**：模拟投资人提问，练习应对尖锐问题。

**怎么用**：
- 上传公司材料 → AI 生成「投资人最容易问倒创始人」的问题（财务/法务/业务/技术/团队/竞争六个维度）
- 录音回答 → AI 评分并指出遗漏要点和逻辑漏洞
- 高频真实问题自动沉淀进题库，下次复用

---

### 7. NPC 智能助手（页面右下角对话框）

**解决什么问题**：对话录入，说「今天和红杉聊了，他们对估值有顾虑」→ AI 自动更新机构档案。

## 常见问题

**Q：启动报错「找不到 uv 命令」**  
A：运行 `pip install uv`，装完后重新打开命令行

**Q：登录失败「账号密码错误」**  
A：确认 `.env` 里 `FOS_ACCOUNTS` 这行包含你的账号，重启后端服务

**Q：页面打开全白/报错**  
A：看后端命令行有没有红色报错，截图发给王波

**Q：尽调匹配全是红色（全部无匹配）**  
A：先确认 Step 1 扫描完成且扫描了正确的材料文件夹

**Q：Settings 页面的 GitHub Token 要不要填？**  
A：不填也能用所有功能，直接跳过

**Q：路演上传后一直「处理中」**  
A：需要 `DASHSCOPE_API_KEY`，检查 `.env` 里这行有没有填

**Q：我改了代码，测试从 832 变成 820 了，怎么办？**  
A：说明你的改动破坏了某些已有功能，先找到哪 12 个测试失败了再修复，不能提交破坏测试的代码
