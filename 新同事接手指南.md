> 写给：第一次接触这套系统的同事（gk001 专属版）  
> 不需要懂代码，只要能看懂中文、会复制粘贴就行  
> 问题找王波

---

## 这套系统是干嘛的

**仓颉 FOS**（融资作战操作系统）= 帮我们管理融资流程的内部工具。

核心功能一句话：
- 记录我们在跟哪些机构谈（Pipeline 看板）
- 路演结束后自动生成评分报告和情报摘要
- 机构发来的尽调清单，AI 自动帮你在材料库里找对应文件
- 所有数据本地存，不上云（GitHub 同步是可选功能）

---

## 第一步：把代码下载下来

```
git clone https://github.com/bog5d/cangjie-fos.git
cd cangjie-fos
```

> ✅ 这是公开仓库，clone 不需要任何密码或 token。  
> 如果 git 提示输入用户名密码，说明你的 git 配置有问题，不是仓库的问题。

## 第二步：安装运行环境（首次约 10 分钟）

### 需要先装好这两样东西

| 工具 | 用途 | 下载地址 |
|------|------|----------|
| Python 3.11+ | 跑后端 | https://www.python.org/downloads/ ← 安装时勾选 **Add to PATH** |
| uv | Python 包管理器 | 装好 Python 后，命令行运行：`pip install uv` |

### 配置环境变量

```bash
cd backend
copy .env.example .env    # Windows
# 或 Mac/Linux: cp .env.example .env
```

用记事本打开 `backend/.env`，填入：

```
DEEPSEEK_API_KEY=sk-王波给你的key
DASHSCOPE_API_KEY=sk-王波给你的key（没有可以先留空）
FOS_ACCOUNTS=zt001:123456:zt,gk001:123456:gk
```

> ⚠️ **重要**：`FOS_ACCOUNTS` 必须包含**所有**需要登录的账号，多个账号用英文逗号分隔。  
> 如果只填 `gk001:123456:gk`，zt001 将无法登录。  
> 如果你是在**王波那台服务器**（已有 zt001 在用的环境）上配置，直接让王波在已有 `.env` 里加上 `gk001` 那段，不要整体替换。

> **关于 GitHub Token**：  
> 系统里有个可选的「数据同步到 GitHub」功能，需要 token，但**不填也能正常用所有功能**。  
> 看到 Settings 页面有 Token 输入框，直接留空跳过就行。  
> 以后需要开启同步功能再找王波要。

## 系统功能使用说明

### 1. 机构 Pipeline 看板（主页左侧）

**解决什么问题**：记录我们在跟哪些机构推进，每家到了哪个阶段。

**怎么用**：
- 点「➕ 新增机构」→ 填机构名、选阶段、选热度 → 创建
- 点任意机构卡片 → 弹出编辑框 → 可以修改阶段、记录里程碑（NDA签了没、见面几次、是否立项等）
- 成就墙（页面右侧）会根据里程碑实时更新统计

**9个里程碑节点**（按顺序）：
路演接触 → NDA签署 → 线下交流 → 立项 → 内部尽调 → 外部尽调 → 投决过会 → 协议签署 → 交割

---

### 2. 尽调响应台（顶部导航「📋 尽调响应」按钮）

**解决什么问题**：机构发来一份50条的尽调清单，AI帮你找材料库里对应的文件，不用你一条条手动找。

**怎么用（三步走）**：

**Step 1 — 扫描材料库**
- 填入你的材料文件夹路径（或点「📁 选择」）
- 点「开始扫描」
- 等进度条跑完（大文件夹要几分钟）

**Step 2 — 上传清单**
- 把机构发的尽调清单粘贴到文本框，或上传 Excel/Word/PDF 文件
- 填写机构名（建议填，后面学习飞轮会用到）
- 点「解析 & 开始匹配」→ 等 AI 跑（会显示「匹配中… 20/50 项」进度）

**Step 3 — 审核 & 确认**

表格顶部的「缺口摘要」一眼看出状况：
```
● 已确认 12 条  ·  ● 需人工 8 条  ·  ● 无材料 30 条
确认进度 24%  ████░░░░░░░░░░
```

按颜色处理：
- 🟢 绿色（≥80%置信度）→ 直接点「✓ 一键确认高置信」
- 🟡 黄色（50-79%）→ 看一眼文件名对不对，对了点「✓」
- 🔴 红色（<50%）→ 点「📂 替换」手动指定正确文件，或点「缺」标记这份材料我们没有

按大类折叠浏览（财务/法务/基本情况），不用从头滚到尾。

**确认完** → 填导出路径 → 点「导出文件夹」→ 系统把所有确认的文件复制到导出目录，生成「缺失清单.txt」

---

### 3. 资产台账 / 轻量匹配器（页面中下部「资产台账」区块）

**解决什么问题**：想快速查一下"我们有没有关于XX的文件"，或者"这批需求文本哪些材料能覆盖"。

**怎么用**：
- 在搜索框输入关键词 → 搜索已扫描的材料库
- 「尽调响应台」区块：填机构名 + 粘贴需求文本 → 点「解析并匹配」→ 轻量快速匹配
- 匹配完后可以点「📋 发起正式尽调」升级到完整的三步 DD 流程

---

### 4. 路演评分工作台（顶部「上传路演录音」入口）

**解决什么问题**：路演录音转文字 + AI 分析 + 生成评分报告，不用手动整理。

**怎么用**：
- 上传录音文件（mp3/wav/m4a）
- 填写机构名和参会者
- 等待处理（AI 分析要几分钟）
- 在「工作台」查看评分和风险点，可以编辑补充

> ⚠️ 需要 `DASHSCOPE_API_KEY` 才能使用语音转写。没有 Key 的话这个功能会失败。

---

### 5. 成就墙（主页右侧）

**解决什么问题**：可视化融资进度，一眼看出「我们接触了多少家、签了几份 NDA、有几家在 DD 阶段」。

**怎么用**：不需要手动操作，编辑机构里程碑后自动更新。

数据会实时刷新，不需要刷页面。

---

### 6. NPC 智能助手（页面右下角对话框）

**解决什么问题**：对话式录入，说「今天和红杉聊了，他们对估值有顾虑」→ AI 自动更新机构档案。

**怎么用**：直接在对话框输入，回车发送。系统会提取其中的机构信息和跟进事项并更新数据库。


## 常见问题

**Q：启动报错「找不到 uv 命令」**  
A：运行 `pip install uv`，装完后重新打开命令行

**Q：登录失败「账号密码错误」**  
A：确认 `.env` 文件里的 `FOS_ACCOUNTS=gk001:123456:gk` 这行存在，重启服务

**Q：页面打开全白/报错**  
A：看一下后端命令行窗口有没有红色报错，截图发给王波

**Q：尽调匹配全是红色（全部无匹配）**  
A：先确认 Step 1 扫描完成了，且扫描的是正确的材料文件夹

**Q：Settings 页面要填 GitHub Token，不填行吗？**  
A：行，直接跳过。那是可选的数据同步功能，不影响其他所有功能正常使用

**Q：路演上传后一直「处理中」**  
A：需要 `DASHSCOPE_API_KEY` 才能转写语音，检查 `.env` 文件里这行有没有填
