## 一、这套系统是干嘛的

**仓颉 FOS**（融资作战操作系统）= 帮我们用「1 个人 + AI」把融资里最重、最高频的活儿干掉的内部工具。

一句话理解每个模块：

| 模块 | 它帮你做什么 |
|------|------|
| 🏛️ **机构漏斗（War Room）** | 记录在跟哪些机构谈、各自走到哪一步（Teaser→尽调→签约） |
| 🎙️ **路演复盘** | 上传路演录音 → 自动转写 → AI 评分 + 风险点诊断 + 情报摘要 |
| 🎤 **路演陪练** | 练讲述（AI 当投资人给覆盖率打分）、抗追问（AI 出压力题审你） |
| 📋 **尽调响应台** | 机构发来的尽调清单，AI 在你材料库里逐条找对应文件，红黄绿标注 |
| 📦 **数据包补全** | 对照「标准数据包模板」体检你的材料库，缺啥、需更新啥，AI 帮你起草 |
| 🤖 **豆豆顾问** | 内嵌的融资 AI 顾问，可问它系统状态、要实战建议 |

**核心理念**：AI 出活、人把关。所有"判断"AI 先做一版，**最后一定有一道人工确认**——你来拍板，AI 不替你做主。

## 二、能力与边界（先建立合理预期，免得误判）

### ✅ 它擅长 / 能做的
- 把几百上千份材料**几十秒扫完建索引**，按"需求→文件"做语义匹配（不是只对文件名）。
- 匹配会**读文件正文做精判**，给出红/黄/绿判定 + 原文证据，判错还会自动换候选重判。
- 路演录音→转写→结构化评分报告；答疑/讲述陪练。
- 跨机构复用：A 机构确认过的"需求→文件"映射，B/C 机构能自动沿用。

### ⚠️ 它的边界 / 已知限制（重要）
- **没有有效的 AI Key，几乎所有"智能"功能都跑不了**（解析清单、匹配、评分、陪练）。第一件事就是确认设置里 DeepSeek Key 是绿的。
- **扫描件 / 图片型 PDF / 加密文件**：正文读取有限（加密件需先登记密码），这类文件可能只能靠文件名匹配，准确性下降——这是已知短板，正在补。
- **AI 会出错**：匹配会有错配、年份串号，所以系统**强制人工确认**。绿的也建议扫一眼证据再放行。
- **数字不要全信 AI**：合成材料/评分里 AI 可能编数字，系统有"事实护栏"会剔除无来源数字，但你仍要核对。
- **数据默认本地存**，不自动上云；多台电脑之间靠"共享部署"或 GitHub 同步（可选）流转。

## 四、拿这些场景来试（从易到难，建议按顺序）

> 都是"只读/可丢弃"的体验，放心点，不会弄坏正式数据。

### 场景 1（最简单）· 路演陪练 · 答疑审问
1. 点顶部 **🎤 路演陪练** → 切到「答疑审问」。
2. 随便贴一段公司业务介绍 → 「生成压力测试问题」。
3. 进"面试房间"，挑一题用打字回答 → 「提交评估」，看 AI 给的命中/遗漏/逻辑漏洞。
   **体会**：AI 怎么当投资人审你；每题带"出处"，没出处的题不会出。

### 场景 2 · 数据包补全 · 给材料库做体检
1. 点 **📦 数据包补全** → 用模板下拉选「标准数据包」（也可「✏️ 编辑/另存」改模板）。
2. 填一个本机有一些文件的文件夹路径 → 「开始扫描并分析缺口」。
3. 看**完整度评分**和"已有/需更新/缺失"三态；点缺失项，AI 会问你几个引导问题，你给点零碎信息 → 「AI 合成材料初稿」。
   **体会**：系统怎么帮你"写材料"，以及"必备项缺失"是怎么点名的。

### 场景 3（核心）· 尽调响应台 · 清单→匹配→人工确认
1. 点 **📋 尽调响应** → Step1 填材料库文件夹路径 → 「开始扫描」。
2. Step2 贴一份尽调清单（几条就行，可含"近三年/2021/2022"这种），**在「项目背景/注意事项」里写清楚口径**（如：含两个主体要分开、年份必须精确）。
3. 点「解析清单」→ **这里会先弹「确认 & 澄清」**：AI 把它拿不准的点做成选择题问你（双主体怎么分、要哪一年），你点选 → 「确认，开始匹配」。
4. 看匹配结果：红/黄/绿 + 命中文件 + 原文证据；多候选可展开换选；逐条 ✓确认 / 缺 / 📂替换。
5.（开发者向）Step3 有个不起眼的「🔧 提示词」——点开能看到 AI 各阶段实际用的提示词。
   **体会**：这是系统的主线能力——"解析→澄清→粗筛→读正文精判→人工确认"整条工作流。

### 场景 4 · 机构漏斗 + 豆豆顾问
- 在机构列表点一张卡片，试试编辑机构画像/阶段。
- 找到豆豆（NPC）对话框，问它一句"现在系统状态怎么样"或"我该先跟进哪家机构"。

## 五、遇到问题，怎么反馈才有用

把这几样发给王波（越具体越快定位）：
1. **你在哪一步、点了什么**（如：尽调台 Step2 点解析清单）。
2. **看到的报错原文**（截图整段，别只说"报错了"）。尤其如果是「鉴权失败 / Key 失效」，多半是 Key 问题。
3. **Key 状态**：设置里 DeepSeek 是绿还是红。
4. **能复现吗**：是每次都这样，还是偶发。

> 王波收到后会判断：是 Key/配置问题（给你新 Key），还是真 Bug（交给 AI 修）。
