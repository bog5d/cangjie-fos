# 仓颉 FOS —— 部署到一台共享服务器（真正的数据流转）

> 给非技术同事看的白话版。照着做,团队就能从「各记各的、靠 GitHub 互相抄」
> 升级成「**所有人共用同一份数据,实时、不丢、不用 Token**」。

---

## 0. 先想明白:为什么这样就解决了

现在的毛病:每台电脑各跑一套、各存一份数据,靠 GitHub 当文件柜互相交换 —— 慢、会抄漏、只传摘要。

这套方案:**只在一台常开的机器上跑后端 + 数据库,所有人用浏览器连它。**
数据从此只有一份,谁改谁看见,根本不存在「流转」—— 因为本来就是一份。

> 比喻:不再各记各的账本,改成墙上挂同一块白板,谁写都写在那一块上,抬头就看见。

GitHub 同步**从此可以不用了**(留着当异地备份也行,见第 4 步)。

---

## 1. 选一台「常开的机器」

| 方案 | 适合 | 最低配置 | 备注 |
|---|---|---|---|
| **A1 · 云服务器** | 经常在外/在家也要用 | 2 核 4G,40G 盘(阿里云/腾讯云轻量,约几十块/月) | 哪都能连,要做好登录+加密 |
| **A2 · 办公室常开电脑/NAS** | 基本都在公司 | 任意一台不关机的电脑 | 最省钱最私密,数据不出公司 |

你们是「两者都有 + 3~5 人」——**推荐 A1 一台最便宜的云主机,配合下面【方案二 Tailscale】**,
既能外网用、又私密、还不用折腾域名证书。

系统选 **Ubuntu 22.04 / 24.04** 最省心。

---

## 2. 装 Docker(一次性,复制粘贴即可)

SSH 登上服务器,执行:

```bash
curl -fsSL https://get.docker.com | sh
```

装完验证:`docker --version` 有输出就行。

---

## 3. 拉代码 + 填配置 + 起服务

```bash
# 1) 拉代码
git clone https://github.com/bog5d/cangjie-fos.git
cd cangjie-fos

# 2) 生成配置文件(从模板复制)
cp backend/.env.production.example backend/.env

# 3) 编辑 backend/.env，重点改三处(用 nano 编辑，Ctrl+O 保存、Ctrl+X 退出)
nano backend/.env
```

`backend/.env` 里**必须改对**的三件事:

1. **账号 `FOS_ACCOUNTS`**:每人一个,密码改强;
   **所有人 tenant_id 设成同一个**(比如都用 `team`)——这样大家才共享同一份数据。
   ```
   FOS_ACCOUNTS=wangbo:你的强密码:team,tongshi1:同事强密码:team
   ```
2. **`DEEPSEEK_API_KEY`** 和 **`DASHSCOPE_API_KEY`**:填上(AI 分析 + 录音转写要用)。
3. **GitHub 三行留空**就行(纯共享服务器模式,最简单)。

然后启动:

```bash
docker compose up -d
```

第一次会构建镜像(含编译前端),约 5~10 分钟。完成后查看状态:

```bash
docker compose ps          # 看到 healthy 就成了
docker compose logs -f     # 看实时日志，Ctrl+C 退出
```

到此,服务已经在 **服务器的 8000 端口** 跑起来了。下一步是让同事能连上它。

---

## 4. 让同事连上来(三选一)

### 方案一 · 办公室局域网(最简单,只在公司用)
1. 查服务器内网 IP:`ip addr | grep 192.168`(类似 `192.168.1.50`)。
2. 同事浏览器打开:`http://192.168.1.50:8000` —— 用各自账号登录即可。

### 方案二 · Tailscale(★推荐:在家/出差也能用,私密、零证书)
Tailscale = 给你们几台设备拉一根加密私有网线,不用公网 IP、不用域名、不用证书。
1. 服务器上装并登录:
   ```bash
   curl -fsSL https://tailscale.com/install.sh | sh
   sudo tailscale up
   ```
   记下它给的地址(形如 `100.x.x.x`)。
2. 每位同事的电脑也装 Tailscale(官网下载,登录同一个组织账号)。
3. 同事浏览器打开:`http://100.x.x.x:8000` —— 在公司、在家、出差都能连,且只有你们组能访问。

### 方案三 · 公网域名 + HTTPS(对外正式访问)
前提:有个域名,A 记录指到服务器公网 IP。
1. 编辑 `deploy/Caddyfile`,把 `fos.example.com` 换成你的域名。
2. 用带 HTTPS 的方式启动:
   ```bash
   docker compose -f docker-compose.yml -f deploy/docker-compose.https.yml up -d
   ```
   Caddy 会自动申请证书。同事打开 `https://你的域名` 即可(8000 不再对公网裸露)。

---

## 5. 把现有数据搬过来(可选)

如果某位同事本地已经攒了不少数据,想带到服务器上(否则就是干净起步):

```bash
# 在服务器上,把本地导出的 data 目录灌进容器的数据卷
# (先把那位同事电脑里的 backend/data 整个拷到服务器当前目录下,命名为 seed-data)
docker compose cp ./seed-data/. cangjie-fos:/app/backend/data/
docker compose restart
```

> ⚠️ 只在「初次上线、服务器还是空的」时做一次。之后所有数据都以服务器为准。

---

## 6. 日常维护(都是一行命令)

```bash
# 升级到最新版
git pull && docker compose up -d --build

# 备份数据(定期做!整库就在这个卷里)
docker run --rm -v cangjie-fos_fos-data:/data -v $(pwd):/backup alpine \
  tar czf /backup/fos-backup-$(date +%F).tar.gz -C /data .

# 重启 / 停止 / 看日志
docker compose restart
docker compose down
docker compose logs -f
```

---

## 7. 安全清单(上外网必看)

- [ ] `FOS_ACCOUNTS` 密码已全部改强,没有 123456。
- [ ] `backend/.env` 没有被提交到 git(它含密码,默认已被忽略)。
- [ ] 外网访问走【Tailscale】或【域名 HTTPS】,**不要**把 8000 端口直接裸露到公网。
- [ ] 定期跑第 6 步的备份命令(或挂定时任务),数据无价。
- [ ] 云服务器的安全组:方案二只放 Tailscale;方案三只放 80/443;**别对全网开 8000**。

---

## 一句话总结
选一台常开机器 → 装 Docker → 填好 `.env`(账号同 tenant + 密钥)→ `docker compose up -d`
→ 同事用 Tailscale/局域网/域名连上 → **从此一份数据,实时共享,GitHub 可以退役了。**
