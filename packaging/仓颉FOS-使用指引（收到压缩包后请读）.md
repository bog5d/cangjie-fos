# 仓颉 FOS v0.5.0 — 从收到压缩包到能正常用（全程说明）

> 给内部测试、试点同事的版本说明。按顺序做即可；**不必会写代码**，会解压、会双击、会往记事本里粘贴内容即可。  
> 通读约 8 分钟；**第一次从解压到能打开页面**，多数人在 **20～40 分钟**（主要花在：装 Python、第一次下载依赖）。

---

## ⚡ 最快入门（3步）

1. 解压 zip → 放到 `C:\FOS\`（路径不要带中文、空格）
2. 在 `CangJie_FOS` 里双击 **`填写API密钥_双击我.bat`** → 填入 Key（找王波要）→ 保存
3. 双击根目录 **`点击开始-仓颉FOS.bat`** → 等10秒 → 浏览器自动打开 → 输入账号密码

## 二、压缩包内容

**标准外发包**解压后，同一层目录里**至少**应有：

| 内容 | 作用 |
|------|------|
| `CangJie_FOS\` | 本系统（网页、后端） |
| `AI_Pitch_Coach\` 且其下有 `src\` | 路演评估引擎；**没有它，「豆豆」和录音复盘会报错** |
| 根目录的《使用指引》和「点击开始」 | 本说明 + 快速启动入口 |

若只有 `CangJie_FOS` 没有 `AI_Pitch_Coach`：联系发你包的人**补发完整包**。

---

## 三、推荐路径

建议在 **C 盘**建英文文件夹，例如：`C:\FOS\`，把 zip **完整解压**到 `C:\FOS\`，保证出现：  
`C:\FOS\CangJie_FOS\` 与 `C:\FOS\AI_Pitch_Coach\` **并列**。

**为什么强调这个**：路径带中文或过长时，部分工具链会出编码或权限问题。

---

## 四、填写API密钥（首次必做）

双击 `CangJie_FOS\填写API密钥_双击我.bat`，用记事本打开 `.env` 文件，把收到的 Key 填在 `=` 后面：

```
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxx       ← AI分析必填
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxx      ← 音频转写必填（不转写可先不填）
FOS_ACCOUNTS=账号:密码:账号（找王波要）  ← 登录账号必填
```

保存后，每次启动系统都会读这个文件。

**也可以在界面里填**：启动后点右上角 **⚙️** → 填写并测试 → 保存。

## 五、启动方式

双击根目录 **`点击开始-仓颉FOS.bat`**（推荐），或：
- 在 `CangJie_FOS` 里运行 `一键启动_Python版.bat`
- PowerShell：在 `CangJie_FOS` 里执行 `.\安装并启动.ps1`

浏览器打开 **`http://127.0.0.1:8000/`**（若未自动打开，请手动输入）。

## 七、新功能：🎯 路演分析

> **本版本最大更新**，推荐第一个测试。

主页顶部有「**🎯 路演分析**」紫色按钮（在「复盘上传向导」右边）。

**最快测试方法（文字稿，1分钟出结果）**：

2. 选「📝 粘贴文字稿」，粘贴：

   说话人A：现在ARR大概多少？IRR怎么算？

## 八、容易卡住的地方

| 现象 | 常见原因 | 建议 |
|------|----------|------|
| 登录失败 | 账号密码错误或大小写不对 | 联系王波确认账号；FOS_ACCOUNTS格式：账号:密码:账号 |
| 主页出现黄色/红色提示条 | 缺 AI_Pitch_Coach 或 Key 没填 | 按条内文字处理；或点「🔧 系统诊断」 |
| 黑色窗口一闪就关 | 没装 Python，或路径带中文 | 装 Python 3.11+（勾选 Add to PATH）；路径改 C:\FOS\ |
| 能开网页，录音分析报错 | 双仓没并列，或没 src 目录 | 检查 C:\FOS\ 里是否有 AI_Pitch_Coach\ |
| 首次启动很久没反应 | 在下载依赖（可数百MB） | 保持窗口不关，等 10～20 分钟 |
| 路演分析卡在步骤4 | DeepSeek Key 没填或无效 | 点右上角 ⚙️ 测试密钥 |
| 步骤3没有说话人 | 文字稿没有「说话人X:」前缀 | 用上方示例格式；或直接无标记也可（归为说话人0） |
| 音频转写一直转圈 | 没填 DashScope Key | 先用文字稿测试；音频需要 DASHSCOPE_API_KEY |

仍解决不了：把**黑色窗口全文**或 **`/api/v1/ready` 的JSON** 截给王波，比只说「不行」快很多。
