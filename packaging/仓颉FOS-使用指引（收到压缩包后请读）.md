# 仓颉 FOS — 从收到压缩包到能正常用（全程说明）

> 给内部测试、试点同事的版本说明。按顺序做即可；**不必会写代码**，会解压、会双击、会往记事本里粘贴内容即可。  
> 通读约 8 分钟；**第一次从解压到能打开页面**，多数人在 **20～40 分钟**（主要花在：装 Python、第一次下载依赖）。

---

## 一、你手上的压缩包里应该有什么

**标准外发包**解压后，同一层目录里**至少**应有（名字需一致，不要只解压一半）：

| 内容 | 作用 |
|------|------|
| `CangJie_FOS\` | 本系统（网页、后端） |
| `AI_Pitch_Coach\` 且其下有 `src\` | 路演评估引擎；**没有它，「豆豆」和录音复盘会报错** |
| 根目录的《使用指引》和「点击开始」 | 本说明 + 快速启动入口 |

若只有 `CangJie_FOS` 没有 `AI_Pitch_Coach`：请联系发你包的人**补发完整包**或按指引把 Coach 放到并列目录（见下）。

`.fos_data\` 可以一开始没有；**资产台账**里暂时没数据是正常现象，用了「仓颉资产台账」做「向上扫描」后才会有索引文件。

---

## 二、推荐路径（减少莫名其妙报错）

1. 在 **C 盘**建英文文件夹，例如：`C:\FOS\`（**不要**解压到桌面、下载、带中文用户名的深路径里）。  
2. 把整个 zip **完整解压**到 `C:\FOS\`，保证出现：  
   `C:\FOS\CangJie_FOS\` 与 `C:\FOS\AI_Pitch_Coach\` **并列**。

**为什么强调这个**：路径带中文或过长时，部分工具链会出编码或权限问题，表现成「能启动但一用就错」或「窗口一闪没」。

---

## 三、从收到包到能用的三条路径（选一条即可）

### 路线 A：最快试跑（已经装过 Python 3.11+）

1. 双击根目录的 **`点击开始-仓颉FOS.bat`**（若发版包内已含）。  
2. 若从未配置过密钥：按窗口提示，去 `CangJie_FOS` 里运行 **`填写API密钥_双击我.bat`**，在记事本里把发你的 Key 填在 `=` 后面，保存。  
3. 再次运行 **`点击开始-仓颉FOS.bat`** 或 `CangJie_FOS\一键启动_Python版.bat`。  
4. 浏览器打开 **`http://127.0.0.1:8000/`** 或 **`http://localhost:8000/`**（若未自动打开，请手动输入）。

### 路线 B：用 PowerShell 启动（与 A 二选一）

1. 在 `CangJie_FOS` 文件夹上 **Shift+右键** →「在此处打开 PowerShell 窗口」或「在终端中打开」。  
2. 执行：  
   `.\安装并启动.ps1`  
3. 其余同路线 A 的密钥与浏览器地址。

### 路线 C：还没装 Python

先看 `CangJie_FOS\同事上手指南.md` 里 **「第一步：检查 Python」**，装好并勾选 **Add Python to PATH** 后，再回路线 A 或 B。

## 四、页面上怎么判断「环境真的好了」

打开网页后，注意顶部是否出现**黄色/琥珀色提示条**（系统自检结果）：

- 若提示 **缺 AI_Pitch_Coach 或 Key**：按条内文字或下节「卡住了怎么办」处理，**不要**先反复点上传。  
- 在浏览器直接访问：  
  **`http://127.0.0.1:8000/api/v1/ready`**  
  可看到 JSON 里的 `ok`、`pitch_coach_ok`、`issues` 等；发你包的人排障时可能会请你截这一页或整段文字。

## 五、容易卡住的地方 — 现象 / 原因 / 你可以怎么做

下面按**真实反馈频率**写，**不是**你操作有问题，而是这类软件常见坑，我们预先把对策写清。

| 现象 | 常见原因 | 建议 |
|------|----------|------|
| 解压后**只有**一个 `CangJie_FOS`，没有 `AI_Pitch_Coach` | 发版不完整或下错附件 | 要完整包；或自己把 `AI_Pitch_Coach` 放到**同一层父目录** |
| 黑色窗口**一闪就关** | 没装 Python、或路径/编码问题 | 用 `cmd` 里执行 bat 看报错；装 Python 3.11+ 并加 PATH；路径改 `C:\FOS\` |
| 能开网页，**录音/豆区一用就报 Pitch_Coach 相关错误** | 双仓没并列，或没 `src` | 对照第二节目录结构；或在 `CangJie_FOS\backend\.env` 里设置 `CANGJIE_PITCH_COACH_ROOT=` 绝对路径 |
| **首次启动**很久没反应 | 在下载依赖（可数百 MB） | 保持窗口不关，换网络或手机热点，等 10～20 分钟再看 |
| 录音一直转圈、失败 | 没配语音相关 Key 或网络不通 | 打开 `填写API密钥_双击我.bat` 检查 `DASHSCOPE_API_KEY` 等；看页面 ready 里 dash 相关提示 |
| 资产台账一直空 | 没跑 FSS「向上扫描」或桥目录不一致 | 用仓颉资产台账做扫描；或让发版人确认 `CANGJIE_FSS_DATA_DIR` 与 FSS 写入路径一致 |
| 浏览器 401、接口全拒 | 发版人开启了 `CANGJIE_API_KEY` | 在 `backend\.env` 看是否要求 API Key；前端需配置与之一致的 `VITE_CANGJIE_API_KEY` 后**重新 build**（由发版人做） |
| 只想本机用却希望手机访问 | 默认绑在 `127.0.0.1` | 发版人改为 `--host 0.0.0.0` 并确保内网安全政策允许 |

仍解决不了：把**黑色窗口全文**或 **`/api/v1/ready` 的 JSON** 截给发你包的人，比只说「不行」快很多。

## 六、使用之后还可能遇到的事（有心理预期就不慌）

- **长录音**：分析要几分钟到十几分钟，**不是卡死**；在任务列表里看状态即可。  
- **同时开很多上传**：可能被提示「任务队列已满」——**稍后再试**或减少同时提交。  
- **关黑色窗口 = 关服务**：用完再关，或发版人教你怎么装成后台服务（一般测试版不强制）。
