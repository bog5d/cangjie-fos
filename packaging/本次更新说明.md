## 本次主要更新

### ✅ 尽调响应台全面升级（v0.8.0 核心功能）

这是上几个版本的核心新功能——**AI 辅助机构尽调文件匹配系统**，全面升级：

#### 🔧 问题修复
- **清单5000字截断彻底修复**：之前超过5000字的尽调清单会被静默截断丢失后半部分，现在改为分块处理（4000字/块 + 重叠去重），即使100条以上的长清单也能完整解析
- **大材料库不再超时**：之前材料库超过200个文件时，AI 匹配会因 token 超限而失败；现在每批自动预筛最相关50个文件，无论材料库多大都正常工作
- **关窗口不再丢会话**：之前关掉向导后，已上传解析的数据就丢失了；现在向导打开时会显示最近5条历史会话，点「恢复」即可继续

#### 🆕 新功能
- **批量确认**：Step3 审核界面新增「✓ 一键确认高置信（N条）」按钮，所有置信度 ≥ 80% 的项一键全部确认，不用逐条点
- **手动文件替换**：对于「无匹配」的项，点 📂 按钮后可直接输入文件路径手动指定，确认后该行变为100%匹配
- **机构阶段自动联动**：上传清单时填写机构名称，系统自动将对应机构的 Pipeline 阶段推进到「尽调」
- **GitHub 自动同步**：导出文件夹后，尽调摘要（需求数/确认数/缺失数）自动同步到 GitHub `analytics/{账号}/dd/` 目录

### ✅ Bug 修复（v0.9.0）

- **资产台账中文搜索**：确认资产搜索功能对中文文件名、摘要、标签的匹配完全正常工作，并新增回归测试保证不退步
- **代码质量**：修复 Python 3.12 对 `datetime.utcnow()` 的 deprecation 警告

## 怎么用（尽调响应台）

1. 点击主页「📋 尽调响应」按钮，打开向导
2. **Step 1 — 扫描材料库**：输入你的材料文件夹路径，点「开始扫描」
   - 例：`D:\泰合2025-03\资料包`
   - 扫描完成后自动进入下一步
3. **Step 2 — 上传尽调清单**：
   - 可上传文件（Excel/Word/PDF）或直接粘贴文字
   - 可选填「机构名称」（填了会自动更新 Pipeline 阶段为"尽调"）
   - 点「解析 & 开始匹配」，AI 自动匹配材料
4. **Step 3 — 审核结果**：
   - 🟢 绿色 = 高置信匹配，点「✓ 一键确认高置信」批量通过
   - 🟡 黄色 = 中等置信，人工核对后逐条确认
   - 🔴 红色 = 无匹配，点📂手动输入文件路径
   - 标记「缺」= 该项材料确实没有（会写入缺失清单）
5. **导出**：输入导出目录，点「导出文件夹」
   - 已确认的文件自动复制到目标目录
   - 生成「缺失清单.txt」列出所有标记为缺的项
   - 同步到 GitHub

## 遇到问题

| 现象 | 解决 |
|------|------|
| 启动失败 / 报错 | 双击 `诊断_打不开请运行我.bat`，把生成的诊断文件发给王波 |
| 登录账号密码不对 | 用 `zt001` / `123456`，或联系王波获取新账号 |
| 上传录音失败 | 确认网络正常（阿里云 ASR 需要联网） |
| AI 分析失败 | DeepSeek 服务可能有延迟，稍等重试；或联系王波 |
| 尽调清单解析后0条 | 检查清单格式——纯文字最兼容，表格建议转存为文字后粘贴 |
| 尽调匹配超时 | 材料库文件夹路径正确且可访问；材料数量无上限，程序会自动预筛 |
