# 仓颉 FOS — 本次更新说明

**版本：0.6.1 | 发布日期：2026-05-15**

---

## ⚠️ 紧急修复：复盘数据未同步到云端

### 问题

通过**向导提交**（点击「提交任务」按钮）完成的路演分析和复盘评分，**没有自动同步到 GitHub 数据备份库**。

具体表现：
- 通过向导提交的任务，`analytics/zt/` 目录下没有新文件
- 切换设备后看不到这些历史记录

### 修复

向导提交流程补加了数据同步步骤，任务完成后会自动推送到 `bog5d/coach_data` 仓库。

---

## 怎么测试

1. 打开 FOS，通过向导提交一次复盘或路演分析
2. 等待 10-30 秒（同步是异步的，后台进行）
3. 打开 https://github.com/bog5d/coach_data/tree/main/analytics/zt/ 确认有新文件

---

## 遇到问题

| 现象 | 解决 |
|------|------|
| 提交后没看到新文件 | 确认 `.env` 里 `COACH_DATA_GITHUB_TOKEN` 已填写（需要 `repo` 权限） |
| 提交失败 / 错误弹窗 | 运行 `CangJie_FOS\诊断_打不开请运行我.bat` |
| 其他问题 | 发给王波 |
