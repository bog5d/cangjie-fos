# 仓颉 FOS 融资作战系统：核心产品战略与工程实施总纲 (MASTER_PRD V3.0)

## 1. 系统灵魂与世界观（禁止偏离）

本系统是一个**「具备自我进化能力的、主动型、游戏化虚拟资本合伙人」**。

- **定位：** 解决融资过程中的组织记忆流失、体力劳动繁重、实战经验断层。
- **核心体验：** 左侧游戏化进度大盘，右侧主动发问的「兜底兄弟」NPC。
- **进化基因：** 系统必须具备「吃一堑长一智」的能力。通过捕捉用户的修正动作，在后台通过模拟对练实现逻辑与话术的自动迭代。

## 2. 目标技术架构：联邦化母子智能体 + 事件驱动 + 自我进化 (Self-Evolution)

1. **大脑引擎 (LangGraph)：** 负责多步骤决策。
2. **感官系统 (Event-Driven)：** 通过 Webhook 和文件监听主动侦测任务。
3. **自我进化飞轮 (The Evolution Flywheel)：**
   - **捕获层：** 实时记录 AI 生成结果与用户手工修改结果的差异 (Diff)。
   - **反思层 (Reflection Node)：** 异步处理这些「失败样本」，提取新的投资人偏好与业务逻辑。
   - **校验层 (Red Teaming)：** 在正式更新前，由系统模拟苛刻投资人进行「对练测试」，验证新逻辑的有效性。
   - **固化层：** 只有通过校验的新逻辑才会被合并进子智能体的核心 Prompt/Skill 库。

## 3. 工程规约与红线

1. **Monorepo 目录规约：** 统一在 `CangJie_FOS` 目录下开发。`/backend`（FastAPI + LangGraph）与 `/frontend`（React/Vite）。
2. **消灭上帝文件：** 严禁任何形式的单体巨型文件。必须严格执行模块化（Router / Service / Schema / Utils / Event）。
3. **迁移铁律：** 迁移旧功能必须带测试迁移。确保 `/backend/tests/` 始终保持 620+ Passed。
4. **数据隐私：** 子智能体数据强绑定 `tenant_id`，禁止敏感明文数据跨租户流动。

## 4. 交付标准

前端编译为 `dist` 由 FastAPI 伺服。通过一键启动脚本运行。
