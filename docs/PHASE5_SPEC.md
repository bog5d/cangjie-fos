# CangJie_FOS Phase 5 SPEC：全感官开启与进化闭环

## A. 核心验收项 (Acceptance Criteria)
- **A1. 眼睛开启 (File Watchdog 实战)：** 激活文件监听。用户将新的“尽调清单”或“BP”拖入指定文件夹，系统需自动触发 ASR/解析流程，并在前端右侧 NPC 面板弹出任务提醒：“检测到新资产，已自动同步至大盘，是否查看？”
- **A2. 耳朵开启 (Webhook 通道)：** 跑通基础的 Webhook 流程。模拟外部 IM 消息（如微信/TG）接入。用户发送指令，后端能唤醒对应的子智能体执行任务并模拟回传。
- **A3. 进化闭环 (Knowledge Injection)：** NPC 的对话逻辑必须真实包含“进化结果”。系统需将 `evolution_guidelines.jsonl` 中的建议动态注入到下一次对话的 System Prompt 中，实现“吃一堑长一智”。
- **A4. 游戏化交互抛光：** - 顶部 Exp 经验条实现平滑增长动画。
  - 增加“成就弹窗”机制：当资料健康度达标或完成高难度复盘时，界面给予视觉奖励。
- **A5. 手动“结算”按钮：** 在 War Room 界面实现“夜间结算”交互，触发 `ReflectionService` 并将进化摘要实时展示给用户。

## B. 自动化测试策略
- 编写“文件移入 -> 任务触发”的端到端 (E2E) 测试。
- 验证注入进化建议前后的对话 Prompt 差异测试。
- 全量测试基线提升至 710+ Passed（含新增感官测试）。
