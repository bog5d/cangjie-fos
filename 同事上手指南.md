# 仓颉 FOS — 同事安装上手指南

> 写给：第一次拿到这套系统的同事  
> 前提：你不需要懂代码，只要会双击和复制粘贴  
> 预计时间：首次约 15~30 分钟（主要是等下载）  
> **当前版本：0.5.0（2026-05-11）**

---

## 一、准备工作：先找王波要这三样东西

- [ ] `CangJie_FOS_Release_xxxxxxxx.zip` 压缩包
- [ ] API Key（一串字母数字，形如 `sk-xxxxxxxx`）
- [ ] **你的专属登录账号密码**（格式：账号:密码，例如 `zt001:123456`）

> **v0.5.0 新增强制登录**：不再可以不输密码直接进系统，必须登录后才能使用。

---

## 二、安装步骤（首次，约 15~30 分钟）

### 第一步：检查 Python 是否已安装

1. `Win + R` → 输入 `cmd` → 回车
2. 输入 `python --version` → 回车
3. 显示 `Python 3.11.x` 或更高 → ✅ 跳下一步
4. 没有或版本太旧 → 去 [python.org/downloads](https://www.python.org/downloads/) 下载，**安装时必须勾选"Add Python to PATH"**

---

### 第二步：解压到正确位置

**⚠️ 不能解压到中文路径（如桌面、下载文件夹）**

推荐：在 C 盘根目录建 `FOS` 文件夹，解压到 `C:\FOS\`

解压后目录结构应如下（AI_Pitch_Coach 由王波单独提供）：

```
C:\FOS\
  ├── AI_Pitch_Coach\       ← 必须，含 src\ 子目录
  ├── CangJie_FOS\
  │     ├── 点击开始-仓颉FOS.bat
  │     ├── 填写API密钥_双击我.bat
  │     ├── 诊断_打不开请运行我.bat
  │     ├── backend\
  │     └── frontend\
  └── .fos_data\            ← 可选，资产台账扫描后自动生成
```

---

### 第三步：填写 API Key

双击 **`填写API密钥_双击我.bat`** → 记事本自动弹出 → 填入 Key → `Ctrl+S` 保存 → 关记事本

```
DEEPSEEK_API_KEY=sk-你的key粘贴在这里
DASHSCOPE_API_KEY=sk-你的key粘贴在这里（ASR转写用，必填）
FOS_ACCOUNTS=账号:密码:账号（找王波要，例如：zt001:123456:zt001）
```

> Key 和账号只填一次，以后直接启动。  
> `FOS_ACCOUNTS` 中，第三个字段（冒号后的账号）决定你的数据分区，与第一个字段保持相同即可。

---

### 第四步：启动

双击 **`点击开始-仓颉FOS.bat`**

- 首次启动会下载 Python 依赖（约5~15分钟，窗口安静是正常的）
- 看到 `✅ CangJie FOS 已启动 → http://localhost:8000` 就成功了
- **浏览器会自动打开，黑色窗口不要关**

---

## 三、系统现有功能一览（0.5.0 版）

### 🔑 登录（v0.5.0 新增）

启动后浏览器显示**登录页**，输入王波给你的账号密码，点「登录」。  
登录后主页顶部显示「👤 你的账号名」即表示成功。

---

### 🎯 路演分析（v0.5.0 全新，推荐先测这个）

**入口：** 主页顶部「**🎯 路演分析**」紫色按钮（在「复盘上传向导」右边）

独立的5步向导，专为路演分析优化：

| 步骤 | 做什么 |
|------|--------|
| 步骤1 | 上传录音（mp3/m4a/wav）**或直接粘贴文字稿**（推荐） |
| 步骤2 | 等待ASR转写进度（文字稿直接跳过此步） |
| 步骤3 | **说话人身份确认** — AI预推测每人角色，你填写真实姓名 |
| 步骤4 | AI深度分析进行中（约30~60秒） |
| 步骤5 | 路演情报报告全文 |

**核心优势**：AI分析时已知道「说话人0是某VC合伙人」，分析质量比旧版高很多。

---

### 🎙 路演录音复盘（核心功能）

**入口：** 主页右上角「**复盘上传向导**」按钮

| 场景 | 操作方式 | 适合谁用 |
|------|----------|---------|
| 有录音文件 | 选择 `.m4a .mp3 .mp4 .wav`，上传分析 | 录音复盘 |
| 有会议文字稿 | 选择 `.txt` 文件，直接上传（跳过转写，更快） | 速记/纪要 |

**分析结果有两种报告，取决于场景选择：**

| 场景选择 | 报告类型 | 看什么 |
|---------|---------|--------|
| `01_机构路演` | 🔍 **路演情报报告** | 对方关键问题、兴趣信号、隐性顾虑、下一步行动 |
| 其他场景（尽调/高管访谈等） | 📊 **风险评分报告** | 总分、风险点列表、改进建议 |

---

### 🔍 路演情报报告（0.4.1 新增）

选择场景 `01_机构路演` 后，AI 不再给你打分，而是像老分析师一样解读整场会议：

**报告包含：**
- **会议氛围**（🔥 高度积极 / ✅ 正常推进 / ❄️ 兴趣不足）+ 阶段判断（初次/深度沟通/准尽调）
- **对方关键问题**：每个问题 + 背后真实关切（核心/关注/礼节三级）
- **兴趣信号**：正面/疑虑/中性，每条附解读
- **隐性顾虑**：对方没说出口但可能担心的
- **关键原声**：最重要的几句原话（值得存档）
- **机构档案更新建议**：AI 基于本次路演建议更新机构画像的内容
- **下一步行动**：谁（我方/对方）、做什么、紧急程度、是否已承诺

---

### 📋 待跟进行动项（0.4.1 新增）

路演情报分析完成后，报告里的「下一步行动」会**自动落盘**，出现在主页的「**待跟进行动项**」清单里。

**在哪里看：** 主页，任务列表下方，点击展开

**怎么用：**
1. 展开清单，看到所有未完成的行动项
2. 确认一项时，点「**完成 ✓**」按钮，该条从清单消失
3. 承诺项（对方/我方已说好的）用橙色标注，建议项用灰色

> 路演回来当天处理完行动项，下次开会前自然不会漏事。

---

### 🏛 机构档案与路演时间线（0.4.1 新增）

**在哪里看：** 主页 → 资料库 → 机构档案 → 点击某机构名 → 侧边栏

侧边栏新增**路演时间线**区块，按时间倒序展示该机构所有关联的路演记录：

- 日期、分析类型（路演/尽调/高管访谈）、状态、路演标题
- 点击任意一条可直接跳转到对应审查台查看详细报告

> 下次见 XX 基金前，先点开他们的档案看一眼历次路演时间线，掌握跟进节奏。

---

### 🤖 豆豆 AI 助手

主页右侧对话框，直接打字问：

- "刚才那场路演对方最担心什么？"
- "见新川基金要准备哪些材料？"
- "我们最近几场路演最常被问到什么问题？"
- "待跟进行动里最紧急的是什么？"

---

### 📁 资料库与匹配

主页下方「资料库」区：

- 上传素材（BP/财务模型/产品文档等）后，系统自动建立索引
- 「智能配对」按钮：输入机构名，自动推荐最匹配的材料组合
- 历次发送记录在「机构档案」里存档

### ✅ 路演分析——文字稿模式（v0.5.0 核心新功能）

**最快测试方法，约1分钟出结果：**

1. 点主页顶部「**🎯 路演分析**」紫色按钮
2. 选「📝 粘贴文字稿」标签
3. 复制下面这段文字粘贴进去：
   ```
   说话人A：你们的退出路径是什么？
   说话人A：现在ARR大概多少？IRR怎么算？基金规模多大？
   说话人B：我们的产品核心壁垒在于AI技术，客户已覆盖500强企业。
   说话人B：商业模式是SaaS订阅制，月ARR约50万，今年目标600万。
   说话人A：你们投资逻辑是什么？我们主要看这个赛道的回报。
   说话人B：我们计划3年内科创板上市。
   ```
4. 路演日期选今天，点「**开始分析 →**」

**步骤3（说话人确认）预期：**
- [ ] 出现2个说话人卡片，AI已推测：A → **GP执行**，B → **企业方创始人**
- [ ] 每个卡片显示「样本台词」（AI推测依据）
- [ ] 填入姓名（随便填），点「**确认身份，开始AI分析 →**」

**步骤5（报告）预期（约60秒后）：**
- [ ] 显示会议温度标签（🔥/☀️/🧊 之一）+ 会议阶段
- [ ] 「核心关注点」列表有至少2条问题
- [ ] 「约定行动项」至少有1条（含负责方和DDL）

### ✅ 路演情报报告（0.4.1 核心新功能）

**测试方法：用文字稿（.txt）上传，最快，不需要等转写**

1. 新建一个文本文件（比如 `test_meeting.txt`），写入：

   ```

   说话人A: 现在ARR大概多少？

   ```

2. 点「复盘上传向导」
3. 场景选 **「01_机构路演」**
4. 被访人填"新川基金——测试"
5. 上传刚才的 `.txt` 文件
6. 提交

**预期结果（约 30~60 秒，文字稿无需转写）：**
- [ ] 任务列表出现"评估中"→"完成"，全程不超过2分钟

- [ ] 报告里有「会议氛围」（hot/warm/cold 之一）和「对方关键问题」
- [ ] 报告里出现至少1条「下一步行动」

**⚠️ 常见问题：**
- 审查台显示「数据异常」→ 重启后端再试一次

## 五、常见问题

| 问题现象 | 解决方法 |
|---------|---------|
| 登录失败/账号不对 | 联系王波确认账号密码；账号区分大小写；检查 `.env` 里 `FOS_ACCOUNTS` 格式 |
| 路演分析步骤3没有说话人 | 文字稿要有「说话人A:」前缀；无标记文本也可（全归说话人0） |
| 路演分析卡在步骤4不动 | 检查 `DEEPSEEK_API_KEY` 是否填写（右上角 ⚙️ 可测试） |
| Windows 弹出安全警告 | 点「更多信息」→「仍要运行」 |
| 黑色窗口一闪就消失 | 右键 `.bat` → 以管理员身份运行 |
| 首次启动卡在下载 | 耐心等 10~15 分钟，或切手机热点重试 |
| 浏览器没自动打开 | 手动输入 `localhost:8000` |
| 路演情报报告一直转圈 | 检查 `DEEPSEEK_API_KEY` 是否正确 |
| 录音上传后卡在转写 | 检查 `DASHSCOPE_API_KEY` 是否填写 |
| 审查台显示「数据异常」 | 重启后端，再次点「查看报告」 |
| 待跟进行动项不出现 | 场景需选 `01_机构路演`，且报告里需有 next_actions |
| 机构时间线没出现 | 参与人确认时需填机构名；或该机构暂无打包记录 |
